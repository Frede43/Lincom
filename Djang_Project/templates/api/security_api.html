Authentication Security

POST /api/security/2fa/configure/

Configure two-factor authentication

Request Body:
{
    "type": "totp",  // or "sms"
    "phone_number": "+257123456789",  // for SMS
    "backup_codes": true,
    "enforce": true,
    "recovery_email": "backup@example.com"
}
Response:
{
    "secret": "JBSWY3DPEHPK3PXP",
    "qr_code": "data:image/png;base64,...",
    "backup_codes": [
        "1234-5678",
        "9012-3456"
    ],
    "status": "enabled"
}
POST /api/security/2fa/verify/

Verify two-factor authentication code

Request Body:
{
    "code": "123456",
    "remember_device": true
}

Access Control

POST /api/security/policies/

Create access policy

Request Body:
{
    "name": "Data Access Policy",
    "description": "Controls access to sensitive data",
    "rules": [
        {
            "resource": "user_data",
            "actions": ["read", "write"],
            "conditions": {
                "roles": ["admin", "data_manager"],
                "ip_range": ["192.168.1.0/24"],
                "time_window": {
                    "start": "09:00",
                    "end": "17:00"
                }
            }
        }
    ],
    "priority": 1,
    "enabled": true
}
GET /api/security/policies/

List access policies

Security Audit

POST /api/security/audit/logs/

Create security audit log

Request Body:
{
    "event": "data_access",
    "user_id": "user-123",
    "resource": "customer_records",
    "action": "read",
    "status": "success",
    "metadata": {
        "ip_address": "192.168.1.100",
        "user_agent": "Mozilla/5.0...",
        "location": "Bujumbura, Burundi"
    },
    "severity": "low"
}
GET /api/security/audit/logs/

Get security audit logs

Query Parameters:
Parameter Type Description
event string Filter by event type
severity string Filter by severity
date_range string Filter by date range
